What’s Included

  • Client theme: templates/darktheme
  • Order form: templates/orderforms/apex_cart
  • Docs: templates/Documentation/

Requirements

  • WHMCS 8+
  • Bootstrap 4, jQuery, Font Awesome (provided by the theme)
  • From darktheme/theme.yaml: bootstrap 4.5.3, jquery 1.12.4, fontawesome 5.10.1
  • From orderforms/apex_cart/theme.yaml: bootstrap 3||4, jquery 1.12, fontawesome 5

1) Installation & Activation

Install the files

  • Upload templates/darktheme
  • Upload templates/orderforms/apex_cart

Activate the client theme

WHMCS Admin → System Settings → General Settings → General tab → Template → darktheme.

Activate the order form

WHMCS Admin → System Settings → Ordering → Default Order Form Template → apex_cart.

Clear template cache

WHMCS Admin → Utilities → System → System Cleanup → Empty Template Cache.

2) File Structure (edit safely)

  • darktheme/header.tpl, darktheme/footer.tpl — Shell, meta tags, Preloader, Navbar, Breadcrumbs
  • darktheme/includes/head.tpl — Styles/scripts includes; custom.css loads last
  • darktheme/components/ — Navbar, plans, sections (domains/features/hosting), shared parts
  • darktheme/css/ — theme.css (core), custom.css (your overrides)
  • darktheme/js/theme.js — Dropdowns, navbar behavior, floating search, sidebar toggles
  • darktheme/THEME_NOTE.MD — Z-index map and template variables
  • orderforms/apex_cart/ — Cart/checkout UI

3) Quick Customizations

A) Logo, favicon, SEO

Logo via {$assetLogoPath} in Navbar. Update in WHMCS General Settings or ensure asset exists.

Favicon/OG image in header.tpl — update if you change paths:

<link rel="icon" href="{$systemurl}/assets/img/favicon.ico">
<meta property="og:image" content="{$systemurl}/assets/img/og-image.jpg">

B) Navbar links and icons

Edit components/sections/shared/navbar/Links.tpl. Menu items come from WHMCS; icons are mapped by label in $svgIcons. Add/change SVG mappings or use the menu item icon class.

C) Homepage plans

Edit components/plans/BasicPlans.tpl:

  • Change product group: {if $group.name eq "Shared Hosting"}
  • Adjust price labels: basicPlanPrice, standardPlanPrice, popularPlanPrice, premiumPlanPrice
  • Ensure product names match your WHMCS products for conditional display

D) Special plans block

Update copy/prices in components/plans/SpecialPlans.tpl.

E) Domain features band

Edit components/sections/domains/DomainsFeatures.tpl. Replace SVGs or text; keep {lang ...} or hardcode text.

F) Feature highlight cards

Update titles and text in components/sections/features/FeaturesSection.tpl.

G) CSS theme overrides

Put your overrides in css/custom.css (loaded last). Tweak colors, spacing, typography, cards, buttons.

H) JavaScript behavior

js/theme.js controls dropdowns, sticky bars, floating search, sidebars. Disable behaviors by removing handlers or scoping with CSS.

4) Language and RTL

RTL enabled when language is arabic. To change texts referenced via {lang key=...}:

  • Use WHMCS language overrides: whmcs/lang/overrides/your-locale.php
  • Or replace {lang ...} with static text in templates

5) Order Form (apex_cart)

  • Activate: Admin → System Settings → Ordering → Default Order Form Template → apex_cart
  • Customize visuals via templates/orderforms/apex_cart/css/ and .../js/
  • Edit layouts in templates/orderforms/apex_cart/*.tpl (e.g., viewcart.tpl, configureproduct.tpl)

6) Advanced Notes

Template variables

See darktheme/THEME_NOTE.MD for common Smarty variables ({$WEB_ROOT}, {$companyname}, {$pagetitle}, {$primarySidebar}, ...).

Performance/UX

  • Preloader: style via custom.css, include via header.tpl
  • theme.js adjusts body padding to prevent content jump

SEO and social meta

Defaults are in header.tpl. Update descriptions, images, and schema.org as needed.

7) Best Practices

  • Back up before changes
  • Prefer custom.css for styling overrides
  • Use language overrides for text
  • Clear template cache after edits
  • Match product names if using conditional price labels
  • Test across devices and in dark mode

8) Troubleshooting

  • Navbar icon missing: label not mapped in $svgIcons
  • Plan price label wrong: product name mismatch with BasicPlans.tpl conditionals
  • Section hidden: some includes render only on homepage or in conditions (see header.tpl)
  • Cache stale: empty template cache and hard-refresh

9) Update Strategy

  • Keep edits small and isolated
  • Document changed files to reapply after updates

10) Quick Checklist
